## 二、逐项差距详析

### GAP-1: "显著的进步"独立判断缺失 [严重]

**知识库规则**（审查指南第二部分第四章2.1）：
创造性包含两个独立要件：(1) 突出的实质性特点（非显而易见性）；(2) 显著的进步（有益技术效果）。两者必须同时满足。

**显著的进步有四种情形**：
1. 效果改善型：与现有技术相比具有更好的技术效果
2. 异途同归型：提供技术构思不同的技术方案，效果基本达到现有技术水平
3. 趋势引领型：代表某种新技术发展趋势
4. 利弊权衡型：某些方面有负面效果，但其他方面具有明显积极的技术效果

**模块现状**：`nodes.go` step3 和 conclusion 仅关注"是否存在技术启示"（突出的实质性特点），没有独立评估"显著的进步"。`InventivenessResult.IsInventive` 直接由三步法第3步结论推导，未区分两个要件。

**影响**：模块可能将"非显而易见但无有益效果"的方案错误判定为具备创造性。

**改进建议**：
- 新增 `step4_significant_progress` 节点或将其融入 `generateConclusionNode`
- `InventivenessResult` 新增 `HasSignificantProgress bool` 字段
- 结论逻辑改为：`IsInventive = NonObvious && HasSignificantProgress`

### GAP-2: 技术问题层次分析法缺失 [中等]

**知识库规则**（审查指南第二部分第四章3.2.1 + 个人笔记）：

| 层次 | 特征 | 示例 |
|------|------|------|
| 表层 | 直接对应区别特征的功能描述 | "如何在设备中增加X部件" |
| 中层 | 对应区别特征在系统中的技术效果 | "如何提高设备的运行效率" |
| 深层 | 对应发明整体方案解决的根本技术难题 | "如何在保证效率的同时降低能耗" |

**模块现状**：`step2DistinguishingFeaturesNode` 的提示词仅要求"重新确定实际解决的技术问题"，无层次分析，容易停留在表层功能描述。

**影响**：技术问题定得过浅→区别特征显得显而易见→错误否定创造性。技术问题定得过深→区别特征显得非显而易见→错误肯定创造性。

**改进建议**：step2 提示词增加层次分析要求，要求 LLM 先区分三层再选择最合适的中层问题作为"实际解决的技术问题"。

### GAP-3: 技术启示判断不够精细 [严重]

**知识库规则**（审查指南 + 复审无效推理模式）：

#### 3a. 三种情形区分不完整（个人笔记扩展为五种）

审查指南定义三种 + 个人笔记补充两种：
1. 区别特征为**公知常识**
2. 区别特征为**同一对比文件其他部分**披露且作用相同
3. 区别特征为**另一份对比文件**披露且作用相同
4. **（个人笔记新增）** 其他对比文件披露了**功能类似但形式不同**的技术手段，可通过公知变化或原理改型获得
5. **（个人笔记新增）** 出于解决**领域公认问题或满足普遍需求**（更便宜、更快捷、更耐久等）的动机

**模块现状**：step3 提示词合并了三种情形，没有区分"同一文件不同部分"和"另一份文件"的不同证据标准。

#### 3b. 反向教导（reverse teaching）缺失
知识库明确：对比文件给出反向教导或存在结合障碍 → **不存在技术启示**。
模块 step3 提示词未包含此判断标准。

#### 3c. 跨领域结合规则缺失
知识库明确：跨领域结合需要有更充分的理由。
模块未区分同领域和跨领域结合。

#### 3d. 改进动机分析缺失
复审无效推理模式中有专门的"改进动机"分析维度：
- 发现技术问题的难易程度
- 不同现有技术结合的动机
- 技术发展趋势的引导作用

**模块现状**：step3 提示词仅笼统要求判断"本领域技术人员是否有动机"，缺少动机分析的具体维度。

**改进建议**：
- step3 提示词重写，明确三种情形+反向教导+跨领域标准
- 新增 `ImprovedMotivation` 分析字段
- 参考复审无效数据的统计权重指导 LLM 判断

### GAP-4: "本领域技术人员"定义未嵌入 [中等]

**知识库规则**（审查指南第二部分第四章2.4）：
"本领域技术人员"是假想的"人"，具有以下属性：
- 知晓申请日/优先权日之前所属技术领域**所有的普通技术知识**
- 能够获知该领域中**所有的现有技术**
- 具有应用该日期之前**常规实验手段**的能力
- **不具有创造能力**
- 如技术问题促使在其他技术领域寻找技术手段，也具有从该其他技术领域获知的能力

**模块现状**：step3 提示词中使用了"本领域技术人员"一词但未定义。LLM 可能使用自己训练数据中不准确的"本领域技术人员"概念。

**改进建议**：将定义嵌入 step3 的 LLM 提示词（类似 `defaultA223Framework()` 已做但未嵌入节点提示词）。

### GAP-5: 五种技术问题情形未区分 [中等]

**知识库规则**（审查指南第二部分第四章3.2.1.1）：

| 情形 | 说明 | 模块覆盖 |
|------|------|---------|
| 情形一 | 最接近现有技术 = 申请人描述的现有技术 → 技术问题同说明书 | ❌ |
| 情形二 | 最接近现有技术 ≠ 申请人描述的现有技术 → 重新确定（最常见） | ✅ 部分 |
| 情形三 | 同领域但方案差异大 → 问题层次/方向可能有较大差异 | ❌ |
| 情形四 | 区别特征为多个且功能上相互支持 → 整体考虑，确定统一问题 | ❌ |
| 情形五 | 所有技术效果均相当 → 问题为"提供一种不同的可供选择的技术方案" | ❌ |

**改进建议**：step2 提示词增加五种情形判断逻辑。

### GAP-6: 六种发明类型特殊规则全部缺失 [严重]

**知识库规则**（审查指南第二部分第四章4.1-4.6）：

| 发明类型 | 关键判断因素 | 模块现状 |
|---------|-------------|---------|
| **开拓性发明** | 技术史上无先例 → **原则上均具备创造性** | ❌ 缺失 |
| **组合发明** | 协同作用（1+1>2）vs 简单叠加（功能无相互作用） | ❌ 缺失 |
| **选择发明** | 预料不到的技术效果，是否可从已知范围直接推导 | ❌ 缺失 |
| **转用发明** | 领域远近、是否克服原领域未曾遇到的困难 | ❌ 缺失 |
| **已知产品新用途** | 是否利用新发现的性质并产生意外效果 | ❌ 缺失 |
| **要素变更** | 关系改变/替代/省略是否产生意外效果或保留全部功能 | ❌ 缺失 |

**影响**：这是最大的功能缺口。例如：
- 一个"组合发明"被分析时，模块不会检查是否存在**协同作用**
- 一个"选择发明"被分析时，模块不会区分是"常规选择"还是"产生意外效果的窄范围选择"

**改进建议**：
- `InventivenessInput` 新增 `InventionType` 字段（pioneering/combination/selection/transfer/new_use/element_change）
- `nodes.go` 新增 `classifyInventionTypeNode` 可选节点
- 根据发明类型调整 step1-3 的提示词和分析重点

### GAP-7: "无贡献特征"规则缺失（2023年局令第84号） [严重]

**知识库规则**（审查指南2023修订第二部分第四章3.1.1）：
- 对技术问题的解决**没有作出贡献**的特征，即使写入权利要求中，**通常也不会对技术方案的创造性产生影响**
- 四维度认定标准：与技术问题的关联、技术效果、常规性、可获知性
- 这是2023年修订新增的重要规则

**模块现状**：完全未处理。模块将所有输入特征一视同仁进行分析。

**改进建议**：
- step2 新增无贡献特征识别子步骤
- `Step2Result` 新增 `NonContributingFeatures []string` 字段
- 结论中排除无贡献特征对创造性判断的影响

### GAP-8: 审查员常见错误仅防范1/4 [中等]

**知识库规则**（审查指南）列出4种常见错误：
1. **"事后诸葛亮"** ← 模块已防范 ✅
2. **技术特征割裂**：将功能上相互支持的特征拆分为独立特征逐一评价 ← 模块未提及 ❌
3. **"本领域技术人员"标准把握不当**（过高/过低/跨领域判断失当）← 模块未防范 ❌
4. **忽视技术效果的整体性**：忽略特征之间的协同作用和整体技术效果 ← 模块未防范 ❌

**改进建议**：各步骤提示词增加对应的防范指令。

### GAP-9: 发明构思分析缺失 [中等]

**知识库规则**（专利实务/创造性-发明构思对技术启示判断的影响）：
- 发明构思体现在"技术问题→解决思路→技术手段"的脉络中
- 发明构思与现有技术构思迥异甚至工作原理相反 → 难以认定存在技术启示
- 五步分析法：提炼专利构思 → 找到对比文件构思 → 比较本质差异 → 分析是否构成结合障碍 → 综合判断

**模块现状**：完全未实现。step3 仅比较技术特征层面的相似性，不分析构思层面的差异。

**改进建议**：step3 提示词增加发明构思分析维度。

### GAP-10: 分析推理与有限试验边界缺失 [中等]

**知识库规则**（专利实务/创造性-分析推理与有限试验）：
- **分析推理**：推理链条必须严密，公知常识对推理能力有重要影响
- **有限试验**：不专指数量，综合考量试验手段/难度/强度
- 关键区分：
  - 少数可选方案+公知验证手段 → 有限试验（不具备创造性）
  - 大量可能组合+无指引缩小范围 → 非有限试验（可能具备创造性）

**模块现状**：未处理此维度。

### GAP-12: 特殊领域规则缺失 [中等]

### GAP-13: 用途限定的影响规则缺失 [低]

**知识库规则**（复审无效推理模式-用途限定的影响）：
- 仅在于使用环境或用途的限定，未带来结构/组成/方法改变 → 不影响创造性判断
- 用途限定隐含产品具有特定结构或性能 → 可能在创造性判断中予以考虑
- "同类性"前提：对比前应先证明同类性、作用一致性

### GAP-14: 改进动机子维度缺失 [中等]

**知识库规则**（专利实务/创造性-改进动机 三篇）：
- 发现技术问题的难易程度（原因超出技术水平→无动机；原因公知→有动机）
- 不同现有技术结合的动机（无法预见结合结果→无动机；破坏原有系统→无动机）
- 技术发展趋势的引导作用（长期排斥→无动机；普遍需求+明确指引→有动机）

### GAP-15: 发明形成过程三种出发点缺失 [中等]

**知识库规则**（个人笔记/创造性-理解发明创造的形成过程）：

发明形成的三种出发点（对三步法第2步有直接影响）：
1. **新的构思或尚未认识的技术问题**：问题本身超出本领域技术人员能力，但一经提出解决手段是显而易见的（此时**仍具备创造性**——即使解决手段显而易见）
2. **为已知技术问题设计新的解决手段**：最常见的类型
3. **发现已知现象的内在原因**：从已知现象出发反推原因

**模块现状**：完全没有区分发明形成的出发点类型。对于第1种情况（问题本身即有创造性），模块可能因"解决手段显而易见"而错误否定创造性。

**改进建议**：step2 增加发明出发点类型识别，对第1种类型调整分析逻辑。

---

### GAP-16: 技术特征划分规则缺失 [中等]

**知识库规则**（个人笔记/三步法-确定技术问题）：
- 互不依存、各自解决不同技术问题的特征 → **应拆分**分别处理
- 紧密联系、协同作用的特征 → **应整体考虑**

这直接影响 step2 中"区别特征"的认定方式。

**模块现状**：所有特征一视同仁，未区分独立特征和协同特征。

---

### GAP-17: "预料不到的技术效果"定位澄清缺失 [中等]

**知识库规则**（个人笔记/预料不到的技术效果）：

关键规则：**预料不到的技术效果是创造性的充分条件，而非必要条件**。
- 具有该效果 → 具备创造性
- 不能以"不具有"为由得出不具备创造性的结论

**模块现状**：`framework.go` 将其列为"辅助判断因素"，但未澄清其"充分非必要"的性质。可能导致 LLM 错误地以"无不具有预料不到效果"来否定创造性。

**改进建议**：`framework.go` 和 conclusion 提示词中明确标注"充分非必要条件"。

---

### GAP-18: 化学领域子类规则细化缺失 [中等]

**知识库规则**（审查指南第二部分第十章 + 个人笔记）：

化学领域覆盖 7 个子类：
| 子类 | 核心判断规则 |
|------|-------------|
| 化合物 | 结构差异→用途/效果→技术启示；结构接近+效果可预期→无创造性 |
| 晶体化合物 | 已知化学结构→须产生预料不到效果 |
| 基因/单克隆抗体 | 已知结构基因的天然突变→无创造性；已知抗原的单抗→通常无创造性 |
| 药物组合物 | 现有技术是否有联用指引、是否存在技术障碍或偏见 |
| 制备方法 | 围绕技术问题梳理多个区别特征确定评判重点 |
| 制药用途 | 新适应症与已知适应症的关系 |
| 中药组合物 | 药味药量变化→仅遵循一般组方规律→无创造性 |

特殊规则：
- **电子等排体**（NH2/CH3、-O-/-NH-、-S-/-O-等）：仅置换且效果相当→无创造性；但产生预料不到效果→仍有创造性
- **已知必然趋势**：效果是已知必然趋势导致的→无创造性（如烷基链增长杀虫效果递增）

---

### GAP-19: 中药领域规则缺失 [中等]

**知识库规则**（审查指南第二部分第十一章）：

中药创造性判断以**"理、法、方、药"**和**"君臣佐使"组方结构**为核心：

| 类型 | 规则 |
|------|------|
| **药味增减** | 公知药对配伍→无创造性；无增减启示+有益效果→有创造性 |
| **药味替换** | 已知功效相同替代+无预料不到效果→无创造性 |
| **药量加减** | 不改变组方结构+常规加减+无预料不到效果→无创造性 |
| **合方发明** | 简单叠加+效果加和→无创造性；无合方启示+有益效果→有创造性 |
| **自组方发明** | 无已知方基础，需记载组方原则和实验数据 |

### GAP-20: 实证统计数据缺失 [低]

**知识库规则**（复审无效/创造性-无效决定裁判规则分析）：

- 39,496 份复审/无效决定中，12,798 份涉及创造性（32.4%）
- 有明确结果的 10,486 份中，**98.9% 认定全部无效**（样本偏向无效请求人）
- 决定要点关键词频率：最接近的现有技术(44.8%)、区别技术特征(30.4%)、技术启示(13.8%)
- 复审委员会高度依赖三步法框架进行说理

可增强模块置信度评估的统计参考。
