# P3 专家盲测单案用例卡：绞车带轴

> 状态：可执行草案
> 适用阶段：P3 专家盲测先导（pilot）或首批正式案件
> 关联总方案：`docs/design/p3-blind-test-plan.md`

---

## 1. 用例定位

本用例用于验证 Mady 在“交底书 + 对比文件 -> 权利要求书草案”场景下的真实可用性，重点考察系统是否能够输出一份**专利代理人可接手修改**的初稿，而不是只做材料摘要或泛化问答。

该案件适合作为首批盲测样本，原因如下：

- 输入材料完整，具备交底书和对比文件，能覆盖“方案理解 -> 区别点提炼 -> 权利要求构造”主链路；
- 输出目标明确，要求同时生成`发明`和`实用新型`两套权利要求书，便于区分不同保护策略；
- 已确认完成脱敏，符合 `AGENTS.md` / `SECURITY.md` 对测试数据的要求；
- 难度处于中高档，既能测稳定性，也能测上限。

## 2. 案件信息

| 项目 | 内容 |
|------|------|
| 用例编号 | `blind-patent-claim-01` |
| 案件代号 | `绞车带轴` |
| 本地样本目录 | `{案件包目录}` |
| 任务类型 | 交底书驱动的权利要求起草 |
| 预期输出 | 发明权利要求书草案 + 实用新型权利要求书草案 |
| 建议难度 | 中高 |
| 建议评审人数 | 2 名专家起步，最好 2-3 名 |
| 建议用途 | pilot 首案 / 正式盲测主样本 |

## 3. 盲测目标

本案要回答以下问题：

1. 系统能否从交底书中稳定抽取必要技术特征，而不是遗漏核心结构或步骤。
2. 系统能否利用对比文件识别区别技术特征，并把区别点转化为可落到权利要求中的保护要点。
3. 系统能否区分`发明`与`实用新型`的撰写策略，而不是简单复制一版文本后做表面改写。
4. 系统输出是否达到“专家愿意基于此继续修改，而不是推倒重写”的初稿质量。

## 4. 输入材料清单

由组织者在盲测前准备以下材料，并放入同一案件包：

- `技术交底书`
- `对比文件` 1 份或多份
- `任务书`（给专家可见）
- `参考答案与评分要点`（仅组织者和分析员可见，专家不可见）

建议在案件包中固定以下命名，降低执行歧义：

- `01-交底书.*`
- `02-对比文件-1.*`
- `03-对比文件-2.*`（如有）
- `90-任务书.md`
- `99-评分要点.md`

## 5. 给专家的任务书

专家在盲测中只看本节，不看系统实现细节，不看参考答案。

### 5.1 任务说明

请基于案件包中的交底书和对比文件，完成以下交付：

1. 起草一套`发明`权利要求书草案；
2. 起草一套`实用新型`权利要求书草案；
3. 用不超过 300 字说明两套方案各自抓取的核心保护点；
4. 用不超过 300 字概述相对对比文件的主要区别技术特征。

### 5.2 输出要求

- 每套至少包含 1 条独立权利要求；
- 从属权利要求数量不强制，但需体现清晰层次；
- 术语需前后一致，不得混用结构名称；
- 不要求一次成稿到可直接提交国知局，但必须达到“代理人可在其上继续修订”的水平；
- 若材料不足以支撑某项特征写入权利要求，应明确标注“不足以写入”的原因，不得自行编造。

### 5.3 时间预算

- 建议总时长：60-90 分钟
- 其中：
  - 阅读材料：15-20 分钟
  - 起草输出：35-50 分钟
  - 自检与补充说明：10-20 分钟

## 6. 组织者操作说明

### 6.1 前置准备

- 确认样本目录内材料已脱敏且文件可正常打开；
- 统一导出为便于阅读的格式（优先 PDF / Markdown / TXT）；
- 准备一份案件包副本给专家，避免专家直接在原始目录中编辑；
- 如使用 Mady 产出作为被测对象，需提前固化模型版本、prompt 版本和运行参数。

### 6.2 建议流程

建议沿用 `docs/design/p3-blind-test-plan.md` 的“先独立、后标注”两阶段流程：

1. `T1 独立作答`
   - 专家在看不到 Mady 产出的前提下，根据案件包独立完成一次起草；
   - 记录起止时间和自评难度。
2. `T2 盲化标注`
   - 向专家展示 Mady 对同一案件的输出；
   - 专家对输出整体做“可采纳 / 需大修 / 不可用”判定；
   - 再按评分表逐项打分并记录修改意见。

若先导阶段人手有限，也可采用简化版：

- 只执行 `T2`，即直接让专家评审 Mady 输出；
- 但该版本只能得到“质量判断”，不能得到完整的人效对照数据。

## 7. 评分表

建议使用 1-5 分制，3 分为“基本可接受但需要人工明显修整”，5 分为“质量高、修改量小”。

| 维度 | 说明 | 分值 |
|------|------|------|
| 技术理解准确性 | 是否正确抓住交底书的核心方案、关键结构与动作关系 | 1-5 |
| 区别点提炼能力 | 是否能结合对比文件识别主要区别技术特征 | 1-5 |
| 发明独立权利要求质量 | 保护主题是否稳定、必要技术特征是否完整 | 1-5 |
| 实用新型独立权利要求质量 | 是否聚焦结构特征，保护边界是否合理 | 1-5 |
| 从属权利要求层次 | 是否形成合理的递进展开，而非堆砌特征 | 1-5 |
| 双案型区分度 | 发明与实用新型是否体现不同撰写策略 | 1-5 |
| 术语一致性与可读性 | 是否存在术语漂移、表达歧义、句式混乱 | 1-5 |
| 可交付性 | 是否达到“代理人可接手修改”的初稿水平 | 1-5 |

总分建议换算规则：

- `34-40`：可直接纳入“通过”样本
- `26-33`：可用但需修整，归为“观察”
- `<=25`：不通过，需回炉分析

## 8. 结果判定

除总分外，还应增加一项整体判定：

| 判定 | 含义 |
|------|------|
| 可采纳 | 可作为初稿进入正式撰写流程，人工仅需常规修订 |
| 需大修 | 方向基本可用，但保护范围、结构层次或术语存在明显问题 |
| 不可用 | 存在核心技术特征遗漏、区别点失真、案型混淆或需要推倒重写 |

建议把以下情形直接视为重大缺陷：

- 核心创新点未进入独立权利要求；
- 发明与实用新型两套文本实质上高度重复，仅做表面改写；
- 相对对比文件的主要区别点未体现，导致权利要求明显撞车；
- 术语混乱到无法判断保护对象；
- 凭空补入材料中不存在的关键技术特征。

## 9. 观察记录

组织者在现场应重点记录以下信息：

- 专家首先卡在哪一步：理解方案、判断区别点，还是构造权利要求；
- 专家对 Mady 输出的第一反应：可信、不稳、不可用，还是“能省一半时间”；
- 专家修改最集中的区域：独权、从权、术语，还是区别点说明；
- 专家是否反复指出同一类问题，例如“漏核心特征”“保护范围太散”“实用新型写得像发明”；
- 专家最终愿不愿意把该输出作为真实案件初稿的起点。

## 10. 通过标准

作为单案通过线，建议使用以下最小标准：

- 至少 2 名专家参与评审；
- 平均总分 `>= 30/40`；
- `可采纳 + 需大修` 占比达到 `100%`，即不得出现“不可用”；
- “双案型区分度”平均分 `>= 3`；
- “可交付性”平均分 `>= 3`；
- 不出现“核心创新点未进入独立权利要求”的重大缺陷。

若用于 pilot，可适度放宽：

- 允许出现 1 次“需大修”；
- 但若出现“不可用”或重大缺陷，必须先修复方法或工作流，再进入正式盲测。

## 11. 后续动作建议

本案盲测完成后，建议按以下方式归因：

- 若主要问题集中在“遗漏关键技术特征”，优先排查交底书理解与特征抽取环节；
- 若主要问题集中在“区别点没抓住”，优先排查对比文件解析与 prior-art 利用方式；
- 若主要问题集中在“发明 / 实用新型写法混淆”，优先补充双案型专用提示词或模板；
- 若主要问题集中在“术语不稳、文本不顺”，可单独做输出重写层优化，但不应掩盖上游理解错误。
