# Mady 数据处理与隐私规范

> 本文档定义 Mady 项目中数据的分类标准、处理原则、存储策略和隐私保护规范。
> 适用于所有参与 Mady 开发的贡献者。

---

## 1. 数据分类

| 类别 | 定义 | 示例 | 允许的存储位置 |
|------|------|------|---------------|
| **生产数据** | 来自真实案件、当事人、专利局的实际业务数据 | 真实交底书、客户信息、案件文档 | 仅限用户本地 `~/.mady/workspace/`，不得提交至仓库 |
| **测试数据** | 人工构造或经脱敏处理的模拟数据 | 示例交底书、mock 案件、合成法律文书 | 可提交至 `integration/testdata/`、`example/` 等目录 |
| **基准数据** | 公开的评估基准题集 | 专利考试真题、《专利审查指南》公开内容 | `agentcore/evaluate/benchmark/testdata/` |
| **配置数据** | Agent Manifest、规则定义、技能文件 | `manifests/`、`domains/rules/`、`skills/` | 可提交至仓库（不含密钥） |

### 数据分类原则

- **生产数据禁止进入版本控制**：任何包含真实当事人信息、案件文档、内部业务逻辑的数据不得提交到仓库
- **测试数据必须脱敏**：使用占位符（如 `张三`、`某公司`、`CN123456789A` 等）替换真实信息
- **基线数据需注明来源**：`agentcore/evaluate/benchmark/` 中的真题数据须在文件头部标注来源

## 2. 敏感信息处理

### 2.1 不得提交到仓库的内容

- API 密钥、Token、密码等凭证（使用环境变量注入，详见 §3）
- 真实案件当事人的姓名、联系方式、地址
- 内部案件编号与真实案件的对应关系
- 超过 500KB 的测试数据文件（pre-commit 钩子已配置 `check-added-large-files`）

### 2.2 测试数据脱敏要求

```
❌ 真实信息:  华为技术有限公司 就专利 CN113014292B 提起无效宣告请求
✅ 脱敏后:    【请求人】就专利 CNXXXXXXXXXB 提起无效宣告请求
```

### 2.3 法律专业数据的特殊要求

- 涉及未公开专利申请内容的测试数据，须额外确认信息来源合规
- 裁判文书数据使用中国裁判文书网已公开内容，标注案号
- 专利数据使用已公开/已公告的专利文献，标注公开号

---

## 3. 密钥与凭证管理

- 所有 API 密钥/Token 必须通过环境变量注入（详见 `README.md`「环境变量」表）
- `.env` 文件已被 `.gitignore` 忽略，用于本地开发配置
- 测试代码中使用占位符（如 `sk-your-test-key`、`test-token`）
- 配置文件中的密钥通过 `pkg/agentconfig` 统一管理，禁止硬编码

---

## 4. 数据存储与保留

| 数据类型 | 默认存储位置 | 保留策略 |
|---------|-------------|---------|
| 会话数据 | `~/.mady/sessions/`（JSONL 格式） | 用户自行管理，不自动删除 |
| 案件工作区 | `~/.mady/workspace/` | 用户自行管理 |
| 知识库 | `~/.mady/knowledge/`（SQLite） | 用户可重建 |
| 评估结果 | `docs/evaluation-baseline-*.md` | 版本控制管理 |
| 日志 | `~/.mady/logs/`（如有） | 滚动保留 |

---

## 5. 合规检查清单

### 提交前检查

- [ ] 未包含真实 API 密钥或凭证
- [ ] 未包含真实案件当事人信息
- [ ] 测试数据已脱敏处理
- [ ] 新增的测试数据文件不超过 500KB
- [ ] 如包含公开来源数据，已标注来源

### 发布前检查

- [ ] 二进制文件未嵌入测试/生产数据
- [ ] 默认配置不含指向内部服务的 URL
- [ ] 日志输出不包含敏感信息
